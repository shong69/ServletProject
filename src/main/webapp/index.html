서버 제출 테스트1

        이름과 나이를 입력 받아서 서버(Java 코드)로 제출

        * 준비물 *
        1) form 태그 : 내부에 작성된 input값을 지정된 경로로 제출해줌
          -action : 제출할 경로
          -method : 제출 방법(GET/POST) 지정. 기본값은 GET
                    (제출 : 클라이언트가 서버로 요청)
        2) input 태그 : 값을 입력하는 양식을 제공함
          -※name : 서버로 제출 시 key 역할을 함
          -입력된 값 또는 value : 서버 제출 시 value 역할을 함
        
        -> ?inputName=홍길동  으로 서버에 넘어감
        

        -----------------------------

        **GET : Get을 통한 요청은 URL 주소 끝에 파라미터로 포함돼 전송됨
        이 부분을 query String이라고 부른다
        -URL 끝에 "?"를 붙이고 변수명1=값1&변수명2=값2...를 붙인다.
        ex) www.example.com/?name1=value1&name2=value2...
        -> 서버에서는 name1과 name2라는 파라미터 명으로 각각 value1과 2 값을 전달받을 수 있다

        [장점]
        -단순함
        -캐싱 가능(caching, 저장)
            -> 주소 저장(== 북마크, 즐겨찾기, 주소 공유...)
        [단점]
        -글자수 제한이 있음
        -보안에 취약함(비밀번호가 주소에 노출됨)
